# 数据格式说明，如果需要可以自行修改，也可以改平台的接收发送的方式！

## 1、数据自动上报
~~~
上位机上报格式（每个设备可单独配置）：

{
"DeviceId": 123,
"Timestamp": "2026-03-18T08:15:37.2401115Z",
"Data": {
"pm50": 150,  //物模型数据
"pm100": 200,  //物模型数据
"pm200": 20  //物模型数据
}
}


读写一体功能指令：
{
"deviceId": 1,
"action": "write",
"writes": {
"ValveStatus": 1
},
"reads": ["ValveStatus", "FlowRate"]
}
~~~

## 2、写功能
~~~
写入功能指令：

{
"deviceId": 123,
"action": "write",
"writes": {
"pm200": 234  //物模型数据
}
}

回复数据：
会更新该设备的全部点位数据，并且携带回复标志
{
"deviceId": 123,
"timestamp": "2026-03-19T01:02:28.6278244Z",
"data": {
"pm50": 150,
"pm100": 200,
"pm200": 1234
},
"replyTo": true    //回复标志
}
~~~

## 3、读功能：
~~~
（1）读功能（传什么读什么）
{
"deviceId": 123,
"action": "query",
"reads": ["pm200", "pm100"]
}
回复数据：
{
"deviceId": 123,
"timestamp": "2026-03-19T01:04:13.1031536Z",
"data": {
"pm100": 200,
"pm200": 1234
},
"replyTo": true  //回复标志
}

（2）读取该设备的所有物模型数据

如果你不传 reads 字段，或者传一个空数组，网关会默认读取该设备下的所有变量：
{
"deviceId": 123,
"action": "query"
}

返回数据：
{
"deviceId": 123,
"timestamp": "2026-03-19T01:16:04.1237497Z",
"data": {
"pm50": 150,
"pm100": 200,
"pm200": 1
},
"replyTo": true
}
~~~

## 4、写、读一体功能

~~~
（1）写指定数，读指定数据指令：
{
"deviceId": 123,
"action": "write",
"writes": {
"pm200": 1
},
"reads": ["pm200", "pm100"]
}

返回数据：
{
"deviceId": 123,
"timestamp": "2026-03-19T01:07:33.0411154Z",
"data": {
"pm100": 200,    //只展示要查询的数据
"pm200": 1      //写入指令中的数据也会被查询出来
},
"replyTo": true
}


（2）写指定数据，读空指令
{
"deviceId": 123,
"action": "write",
"writes": {
"pm200": 2
},
"reads": []
}

返回该设备全部数据：

{
"deviceId": 123,
"timestamp": "2026-03-19T01:17:15.3440855Z",
"data": {
"pm50": 150,
"pm100": 200,
"pm200": 2
},
"replyTo": true
}
~~~
